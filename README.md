# WEB. Промо-портал и Live-статистика для VR Shooter
## Описание проекта
Разработка комплексного веб-ресурса для существующего LBE (Location-Based Entertainment) VR-шутера. Сайт выполняет три функции:
- Промо (Marketing): Привлечение игроков и продажа франшизы/лицензий владельцам клубов.
- Сервис (Booking): Система бронирования игрового времени.
- Live-статистика (Gamedev integration): Отображение хода матчей в реальном времени и рейтингов (как в киберспорте).
## Целевая аудитория
- Игроки: Хотят узнать об игре, забронировать игру, посмотреть свою статистику.
- Владельцы клубов (B2B): Хотят купить лицензию на игру, посмотреть отзывы коллег.
## Структура и Функционал Сайта
1. Главная страница (Landing Page):
- Hero Block: Эпичный трейлер игры (автовоспроизведение/по клику), заголовок, кнопка "Забронировать".
- Об игре: Описание сеттинга, уникальных механик (Free Roam, PvP, оружие), качественные скриншоты и концепт-арты (слайдер/галерея).
- Социальное доказательство:
  - Отзывы игроков: "Живые" карточки с фото и цитатами.
  - Отзывы партнеров (Владельцев клубов): "Игра повысила выручку на 30%..."
- Карта локаций: Интерактивная карта или список городов/клубов, где уже есть эта игра.
2. Раздел для Бизнеса (B2B):
- Предложение: "Запустите наш шутер в своем VR-клубе". Преимущества, технические требования, монетизация.
- Форма заявки: "Стать партнером" (сбор лидов в CRM или на почту/в телеграм).
3. Система Бронирования (Booking):
- Выбор города и конкретного клуба.
- Календарь с выбором даты и времени (слоты).
- Форма ввода контактов (Имя, Телефон). Примечание: Реальная оплата не требуется, достаточно факта отправки заявки администратору выбранного клуба.
4. Модуль Статистики (Live Ops):
- Интеграция: Прием данных от игрового сервера (по-прежнему нужен Mock-скрипт, эмулирующий посылку JSON-пакетов от "реальной" игры, так как доступа к продову у студентов нет).
- Live Match: Страница, показывающая текущий матч (Карта, Счет команд Red vs Blue, Статус игроков Dead/Alive).
- Global Ranking: Общая таблица лидеров (Никнейм, K/D Ratio, Winrate).
5. Telegram-бот:
- Дублирует функционал статистики (поиск игрока, уведомление о результатах).
- Уведомляет админов о новых заявках на бронирование или покупку франшизы.
## Технический стек
- Frontend: React / Vue.js / Next.js + Tailwind CSS (для быстрого и стильного UI).
- Backend: Node.js (Express/Nest) или Python (FastAPI).
- Database: PostgreSQL или MongoDB.
- Telegram Integration: Telegraf / Aiogram.
## Требования к контенту 
- логотип игры, скриншоты, текст "легенды", видео-трейлер (нарезка стоков или монтаж геймплея аналогов).
## Требования к команде
- Дизайн/Верстка: Создание продающего, современного дизайна (Dark theme, Cyberpunk/Sci-Fi).
- Fullstack dev: Реализация бэкенда для хранения матчей и заявок, создание API для приема статистики.
- Bot dev: Синхронизация бота с базой сайта.
## Результаты
- Полноценный развернутый сайт (Landing + Stats + Booking).
- Админ-панель (упрощенная) для просмотра заявок на бронь.
- Работающий Telegram-бот.
- Эмулятор игрового сервера (скрипт), который "гонит" данные статистики на сайт, чтобы показать жюри "Live-режим".
## Критерии оценки
- Дизайн и UX: Сайт должен выглядеть дорого и продающе.
- Работающая воронка: Кнопка "Забронировать" -> Выбор времени -> Заявка падает в Телеграм админу.
- Live-магия: Данные на странице статистики должны обновляться сами (без перезагрузки страницы).
- Адаптив: Корректное отображение на мобильных.
